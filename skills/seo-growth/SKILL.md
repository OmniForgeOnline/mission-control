# SEO Growth

## When to use

Blog posts, landing pages, SEO investigations, and content needing search visibility. Use for hypothesis definition, seo_review steps, and remediation planning.

## How

The harness attaches **Workspace artifacts** (file list, markdown links, headings, excerpts) before your turn. Start there — do not re-inventory files unless something is missing.

1. State the SEO goal: ranking target, CTR improvement, indexation fix, or content gap.
2. Review on-page elements: title, meta description, H1, canonical, internal links, schema.
3. Check technical blockers: robots.txt, sitemap, crawl errors, Core Web Vitals signals if data exists.
4. Propose specific, measurable changes — not "improve SEO".
5. For blog drafts: suggest title variants, meta description (≤160 chars), and primary keyword placement.

Conversation steps: wrap remediation plans in `<proposed_plan>`. Agent turns: use **Findings**, **Recommendations**, **Priority**.

## Anti-patterns

- Keyword stuffing or hidden text suggestions.
- Recommending changes without inspecting actual page content or URLs.
- Chasing vanity metrics with no tie to business goals.
- Black-hat tactics (cloaking, link schemes).

## Programmatic surface

- `data-analysis` skill — traffic or ranking evidence when available.
- `gbrain_search("seo guidelines")` — team-specific SEO rules.
